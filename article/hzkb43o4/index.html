Docker安装nginx

702 字约 2 分钟

dockernginx

2024-12-26

注意

以下操作都在Ubuntu环境下进行，也可在其他Linux系统下操作，但如果出现问题请自行解决。

注意

使用non-root用户启动容器，默认当前用户，如有需要请自行更改

docker工作目录树

  • docker
    • nginx
      • conf
        • conf.d
          • default.conf
        • nginx.conf
      • Dockerfile
    • compose.yaml
    • .env

新建docker工作目录

mkdir -p docker && cd docker

创建Dockerfile用来自定义构建容器

  1. 新建nginx目录、conf配置目录
mkdir -p nginx && cd nginx
mkdir -p conf/conf.d
  1. 创建nginx.conf和default.conf配置文件

  1. 创建Dockerfile文件
FROM nginx:1.27.3-alpine

ARG UID
ARG GID

RUN chown -R $UID:$GID /var/cache/nginx

创建docker compose配置文件 compose.yaml

注意

  • 数据卷都挂载在 ${HOME}/data/nginx 目录下，请提前创建好对应目录，也可自行修改挂载目录
  • nginx.conf 配置文件需要拷贝到 ${HOME}/data/nginx/conf 目录下
  • default.conf 配置文件需要拷贝到 ${HOME}/data/nginx/conf/conf.d 目录下
mkdir -p ${HOME}/data/nginx
mkdir -p ${HOME}/data/nginx/conf/conf.d
services:
  nginx:
    build: 
      context: nginx
      dockerfile: Dockerfile
      args:
        UID: ${UID}
        GID: ${GID}
    image: nginx:1.27.3-alpine
    container_name: nginx
    restart: always
    # non-root users are not allowed to bind to ports below 1024
    # network_mode: "host"
    user: ${UID}:${GID}
    networks:
      - docker_net0
    ports:
      - 80:80
      - 443:443
    environment:
      TZ: Asia/Shanghai
    volumes:
      # nginx目录挂载
      - ${HOME}/data/nginx:/nginx
      # 配置文件挂载
      - ${HOME}/data/nginx/conf/nginx.conf:/etc/nginx/nginx.conf
      # 配置目录挂载
      - ${HOME}/data/nginx/conf/conf.d:/etc/nginx/conf.d

networks:
  # 自定义一个桥接网络
  docker_net0:

构建并启动

echo -e "UID=$(id -u)\nGID=$(id -g)" > .env
docker compose up --build -d nginx
