安装/使用vuepress-theme-plume主题

点我查看[安装/使用文档] vuepress-theme-plume 是一个基于 VuePress 的主题，无论您是想写生活类博客、技术类博客、或者是产品文档、知识库、系列文档等，主题都可以满足您的需求。 主题针对文本内容、图片内容的表现做了大量的优化，特别是针对Markdown内容的语法做了丰富的扩展，你可以很轻松的利用这些特性编写美观、易读、表现力丰富的内容。

使用github pages部署

我分了两个仓库为静态博客提供服务，一个私有仓库（以下用private代替）用来存放博客源码，一个公开仓库（以下用public代替）用来存放编译后的静态资源文件。

  1. 首先在项目根目录下创建github actionsCI所需的编译构建配置文件（如果在安装时选择了github actions，会自动创建对应文件）。
  • 你的博客项目目录
    • .github
      • workflows
        • deploy.yml
    • xxx
    • xxx

deploy.yml配置可直接复用我的:

name: deploy

on:
  # 每当 push 到 main 分支时触发部署
  # Deployment is triggered whenever a push is made to the main branch.
  push:
    branches: [main]
  # 手动触发部署
  # Manually trigger deployment
  workflow_dispatch:

jobs:
  docs:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4
        with:
          # “最近更新时间” 等 git 日志相关信息，需要拉取全部提交记录
          # "Last updated time" and other git log-related information require fetching all commit records.
          fetch-depth: 0

      - name: Setup pnpm
        uses: pnpm/action-setup@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          # 选择要使用的 node 版本
          node-version: 20

      # 安装依赖
      # Install dependencies
      - name: Install Dependencies
        run: pnpm install --frozen-lockfile

      # 运行构建脚本
      # Run the build script
      - name: Build VuePress site
        run: pnpm run docs:build

      # 部署
      - name: Deploy
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          folder: docs/.vuepress/dist # The folder the action should deploy.
          token: ${{ secrets.BLOG_TOKEN }}
          repository-name: 你的public仓库（如: xxx/xxx.github.io）
          force: true

提示

  • token配置需要你自己在private仓库中设置
  • 注意把repository-name换成你自己的public仓库
  • 当你写完文章使用git push推送到private私有仓库后会自动执行CI流程，该流程会把编译后的静态资源推送到public公开仓库
点我查看在private仓库中添加 token

  1. public公开仓库创建

参考github pages文档创建一个public仓库

注意

理论上创建完public仓库就可以了，private仓库CI工作流会自动为public仓库创建一个gh-pages分支作为发布分支，如果Settings -> Pages -> Build and deployment -> Branch为None，需要你手动选择为gh-pages

如果没有意外，你已成功搭建好你的博客网站，可以自由发布你的文章了。

可选的配置

  1. 添加文章评论功能 点我查看文档

推荐选择Giscus，这个配置最为简单

  1. 自定义域名（需要拥有一个域名），两种方式

一种是直接在public仓库的Settings->Pages中配置，这种方式比较简单，但是由于github pages部署在国外，访问速度很慢（很大概率无法访问）。

如果启用HTTPS的话，需要在你购买域名的平台配置一下DNS解析地址。参考文档，SSL证书github pages会自动帮你安装一个免费的不需要额外购买。

另一种是自己购买一个服务器，使用Nginx或Caddy反向代理https://xxx.github.io。这种方式的好处是可以把Nginx或Caddy当作CDN缓存静态资源网站，使得国内可以加速访问你的博客，但是可能会需要你进行备案和管理SSL证书（如果你使用Caddy的话，不需要自己手动管理证书，它会自动帮你申请安装一个免费的）。可以参考我的Caddyfile配置：

your.domain.com { # 更换为你自己的域名
  encode zstd gzip

  # 反向代理到 GitHub Pages
  reverse_proxy https://xxx.github.io { # 更换
    header_up Host {upstream_hostport}
  }

  # 缓存策略
  @static {
    path *.css *.js *.png *.jpg *.jpeg *.svg *.gif *.ico *.woff *.woff2 *.ttf *.otf
  }

  header @static {
    Cache-Control "public, max-age=3600, immutable"
  }

  # 日志记录
  log {
    output file /var/log/caddy/access.log {
      roll_size 50MiB
      roll_keep_for 240h
    }
    format json
  }
}
  1. 配置百度统计或谷歌统计（可统计博客网站的访问数据）

你可以参考这个网站文档教程去创建一个基础的网站配置

安装百度统计分析插件

pnpm i -D @vuepress/plugin-baidu-analytics@next

在config.ts配置文件中添加以下配置（参考文档）

...
import { baiduAnalyticsPlugin } from '@vuepress/plugin-baidu-analytics'
...
export default defineUserConfig({
  ...
  plugins: [
    baiduAnalyticsPlugin({
      id: 'xxxxxxxxxxxxx' // 换成链接?后面的参数即可 https://hm.baidu.com/hm.js?xxxxxxxxxxx
    })
  ],
  ...
})
