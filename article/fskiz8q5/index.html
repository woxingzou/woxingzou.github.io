Docker安装postgresql

3569 字约 12 分钟

dockerpostgresql

2024-12-28

注意

以下操作都在Ubuntu环境下进行，也可在其他Linux系统下操作，但如果出现问题请自行解决。

注意

使用non-root用户启动容器，默认当前用户，如有需要请自行更改

docker工作目录树

  • docker
    • postgres
      • conf
        • postgresql.conf
      • sql
        • xxx.sql
        • xxx.sql
      • Dockerfile
    • compose.yaml
    • .env

新建docker工作目录

mkdir -p docker && cd docker

创建Dockerfile用来自定义构建容器

  1. 新建postgres目录、conf配置文件目录、sql脚本目录（可选）
mkdir -p postgres && cd postgres
mkdir -p conf
mkdir -p sql # 可选

提示

可以把建表sql脚本放入 postgres/sql 目录下，在Dockerfile中使用 COPY sql/*.sql /docker-entrypoint-initdb.d/ 语句将脚本拷贝到容器内，postgres容器启动时会自动帮你执行。

  1. conf目录下新建配置文件postgresql.conf

提示

vim conf/postgresql.conf
点我查看postgresql.conf配置

提示

配置文件是从postgres容器内拷贝出来的，修改了最大连接数和shared_buffers内存大小，如有需要可自行修改

  1. 创建Dockerfile文件
FROM postgres:17.2-alpine

# COPY sql/ry_*.sql /docker-entrypoint-initdb.d/ # 可选

ARG USER

RUN useradd -m -s /bin/bash $USER

创建docker compose配置文件 compose.yaml

注意

  • 数据卷都挂载在 ${HOME}/data/postgres 目录下，请提前创建好对应目录，也可自行修改挂载目录
  • postgresql.conf 配置文件需要拷贝到 ${HOME}/data/postgres/conf 目录下
mkdir -p ${HOME}/data/postgres/conf
mkdir -p ${HOME}/data/postgres/data
services:
  postgres:
    build: 
      context: postgres
      dockerfile: Dockerfile
    image: postgres:17.2-alpine
    container_name: postgres
    restart: always
    user: ${UID}:${GID}
    networks:
      - docker_net0
    ports:
      - 5432:5432
    environment:
      TZ: Asia/Shanghai
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-admin@123456}
    # 指定配置文件启动
    command: ["postgres", "-c", "config_file=/etc/postgresql/postgresql.conf"]
    volumes:
      # 数据目录挂载
      - ${HOME}/data/postgres/data:/var/lib/postgresql/data
      - ${HOME}/data/postgres/conf/postgresql.conf:/etc/postgresql/postgresql.conf

networks:
  # 自定义一个桥接网络
  docker_net0:

提示

以下环境变量可自行修改

  • POSTGRES_PASSWORD: 登陆密码，必填
  • POSTGRES_USER: 登录用户，选填，默认为postgres
  • POSTGRES_DB: 数据库，选填，默认为postgres

构建并启动

echo -e "UID=$(id -u)\nGID=$(id -g)" > .env
docker compose up --build -d postgres
