VMware-Ubuntu24虚拟机配置静态IP

502 字约 2 分钟

ubuntu

2025-01-12

介绍

我的路由器LAN口IP地址分配方式是通过DHCP自动获取的，每台设备也同样使用DHCP自动获取IP和DNS地址，虽然很方便，但是也带来一点困扰。比如我每次重启虚拟机后就要改一下宿主机上.ssh/config文件中配置的HOSTNAME，ssh很不方便。于是打算给虚拟机分配一个静态IP，查了一些资料，在此做一下记录，防止以后忘了。

查看路由器DHCP地址池

连接路由器的设备被分配的IP都是在DHCP地址池里的，因此我们可以给虚拟机一个地址池范围外的IP，这样就不必担心IP地址冲突了，你可以登陆你的路由器后台查看一下，比如我的局域网IP地址范围是192.168.0.0 ~ 192.168.0.255，路由器DHCP地址池范围是192.168.0.100 ~ 192.168.0.199，那我可选的静态IP范围就在192.168.0.2 ~ 192.168.0.99和192.168.0.200 ~ 192.168.0.254之间。我挑了个192.168.0.88，不为别的，就图它吉利。

修改Ubuntu24网络配置

配置文件在/etc/netplan目录下，文件名是50-cloud-init.yaml，你可以先使用ip addr查看一下你的网卡名称，通常来说都是ens33或ens32。

sudo vim /etc/netplan/50-cloud-init.yaml

配置文件参考

# This file is generated from information provided by the datasource.  Changes
# to it will not persist across an instance reboot.  To disable cloud-init's
# network configuration capabilities, write a file
# /etc/cloud/cloud.cfg.d/99-disable-network-config.cfg with the following:
# network: {config: disabled}
network:
  ethernets:
    ens33:
      dhcp4: false
      addresses: [192.168.0.88/24] # 注意你的IP网段和子网掩码
      # DNS服务器
      nameservers:
        addresses: [8.8.8.8, 4.4.4.4]
      # 默认网关，和你宿主机的默认网关相同就行
      routes:
        - to: default
          via: 192.168.0.1
  version: 2

验证

sudo netplan apply

ip addr # 查看IP是否设置成功

ip route show # 查看默认网关是否设置成功

ping 你的宿主机IP或者外网域名 # 查看网络是否正常
