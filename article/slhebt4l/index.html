Docker安装jenkins

544 字约 2 分钟

dockerjenkins

2024-12-29

注意

以下操作都在Ubuntu环境下进行，也可在其他Linux系统下操作，但如果出现问题请自行解决。

注意

jenkins镜像会创建名为jenkins的用户，因此不需要我们做额外的non-root映射

docker工作目录树

  • docker
    • jenkins
      • Dockerfile
    • compose.yaml
    • .env

新建docker工作目录

mkdir -p docker && cd docker

创建Dockerfile用来自定义构建容器

  1. 新建jenkins目录
mkdir -p jenkins && cd jenkins
  1. 创建Dockerfile文件
FROM jenkins/jenkins:alpine-jdk17

创建docker compose配置文件 compose.yaml

提示

jenkins使用匿名数据卷，不需要在宿主机创建额外的挂载目录

services:
  jenkins:
    build:
      context: jenkins
      dockerfile: Dockerfile
    image: jenkins/jenkins:alpine-jdk17
    container_name: jenkins
    restart: always
    user: ${UID}:${GID}
    networks:
      - docker_net0
    ports:
      - 8080:8080
      - 50000:50000
    environment:
      TZ: Asia/Shanghai
      JENKINS_OPTS: "--prefix=/jenkins"
      JAVA_OPTS: "-Duser.timezone=Asia/Shanghai"
    # 执行命令
    volumes:
      # 数据目录挂载
      - jenkins_home:/var/jenkins_home

networks:
  # 自定义一个桥接网络
  docker_net0:

volumes:
  # 创建一个匿名数据卷
  jenkins_home:

注意

restart: always，一定要设置为always，否则jenkins在线更新后不会自动重启

提示

以下环境变量可自行修改

  • JENKINS_OPTS: 访问前缀，选填
  • JAVA_OPTS: 时区设置为东八区，选填

构建并启动

echo -e "UID=$(id -u)\nGID=$(id -g)" > .env
docker compose up --build -d jenkins
