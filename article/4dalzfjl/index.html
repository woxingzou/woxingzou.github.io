Docker安装minio

416 字约 1 分钟

dockerminio

2024-12-26

注意

以下操作都在Ubuntu环境下进行，也可在其他Linux系统下操作，但如果出现问题请自行解决。

注意

使用non-root用户启动容器，默认当前用户，如有需要请自行更改

docker工作目录树

  • docker
    • minio
      • Dockerfile
    • compose.yaml
    • .env

新建docker工作目录

mkdir -p docker && cd docker

创建Dockerfile用来自定义构建容器

  1. 新建minio目录
mkdir -p minio && cd minio
  1. 创建Dockerfile文件
FROM minio/minio:RELEASE.2024-11-07T00-52-20Z

创建docker compose配置文件 compose.yaml

注意

数据卷都挂载在 ${HOME}/data/minio 目录下，请提前创建好对应目录，也可自行修改挂载目录

mkdir -p ${HOME}/data/minio
services:
  minio:
    build:
      context: minio
      dockerfile: Dockerfile
    image: minio:RELEASE.2024-11-07T00-52-20Z
    container_name: minio
    restart: always
    user: ${UID}:${GID}
    networks:
      - docker_net0
    ports:
      # api 端口
      - 9000:9000
      # 控制台端口
      - 9001:9001
    environment:
      TZ: Asia/Shanghai
      # 管理后台用户名
      MINIO_ROOT_USER: ${MINIO_ROOT_USER:-admin}
      # 管理后台密码，最小8个字符
      MINIO_ROOT_PASSWORD: ${MINIO_ROOT_USER:-admin@123456}
      # https需要指定域名
      # MINIO_SERVER_URL: ""
      # MINIO_BROWSER_REDIRECT_URL: ""
    # 执行命令
    command: server /data -address ":9000" --console-address ":9001"
    volumes:
      # 数据目录挂载
      - ${HOME}/data/minio:/data

networks:
  # 自定义一个桥接网络
  docker_net0:

提示

以下环境变量可自行指定

  • MINIO_ROOT_USER
  • MINIO_ROOT_PASSWORD
  • 如果需要用ngixn反向代理管理后台请填写 MINIO_BROWSER_REDIRECT_URL

构建并启动

echo -e "UID=$(id -u)\nGID=$(id -g)" > .env
docker compose up --build -d minio

访问MinIO Console管理后台

浏览器访问 http://127.0.0.1:9001
